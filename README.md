# HW4: FPGA VGA 顯示與乒乓球遊戲圖形化

## 第一階段：VGA 顯示介面實作

### 程式功能說明
本階段的核心在於建立 FPGA 與螢幕之間的通訊基礎。程式碼主要執行以下工作：

1.  **時脈生成 (Clock Divider)**：
    將開發板的 100MHz 系統時脈進行二除頻，產生 **50MHz** 的像素時脈 (Pixel Clock)，以符合 SVGA (800x600) 的工業標準頻率。

2.  **掃描計數 (Counters)**：
    利用 `h_count` (水平) 與 `v_count` (垂直) 兩個計數器追蹤目前的掃描點位置。
    * 水平計數滿 1056 (Total Pixels) 後歸零，並觸發垂直計數加 1。
    * 垂直計數滿 628 (Total Lines) 後歸零，完成一個 Frame 的掃描。

3.  **同步訊號產生 (Sync Generation)**：
    根據計數器的數值，在特定的時間點拉低 `H_Sync` 與 `V_Sync` 電位，產生螢幕所需的同步訊號，確保影像不會歪斜或滾動。

## 第二階段：乒乓球遊戲圖形化與特效

### 程式功能說明
本階段將原本僅能透過 LED 顯示的乒乓球遊戲邏輯，整合至 VGA 介面，並加入圖形運算與色彩回饋。

1.  **訊號整合 (Integration)**：
    頂層模組 `Top_PingPong_VGA` 實例化了遊戲核心 (`pingpong`) 與 VGA 控制器 (`VGA`)。它將實體按鍵訊號傳入遊戲核心，並將遊戲輸出的燈號狀態 (`led_state`) 與狀態模式 (`is_score`) 傳送給繪圖邏輯。

2.  **圓形繪圖演算法 (Circle Rendering)**：
    捨棄簡單的矩形繪製，程式在掃描過程中即時進行幾何運算。
    * 利用公式 $(x - center)^2 + (y - center)^2 < R^2$ 判斷目前像素點是否位於圓形範圍內。
    * 依據目前的水平位置 `h_pos` 計算出對應的是哪一顆燈號 (Index 0~7)，實現 8 顆燈號的排列顯示。

3.  **動態色彩回饋 (Dynamic Coloring)**：
    為了增強遊戲體驗，程式加入了顏色判斷邏輯：
    * **遊戲進行中**：程式偵測到一般模式，VGA RGB 輸出 **綠色** 訊號。
    * **得分與獲勝**：當偵測到遊戲核心傳來「得分模式」訊號時，VGA RGB 切換為 **紅色** 訊號，提供清晰的視覺狀態提示。
